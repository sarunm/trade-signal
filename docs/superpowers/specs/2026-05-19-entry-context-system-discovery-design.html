2Auto-Fill Entry Context

New file: api/services/entry_context.py — called by trade_logger.py after ENTRY_IN upsert, only for real trades (is_paper=False) with open_price set. Does not commit — caller commits.

_fill_fib_proximity(session, trade)

  1. Query fib_levels for symbol = trade.symbol (most recent row)
  2. If no row → skip (fields stay null)
  3. Collect all label/price pairs from levels dict (key "0.000" → label "PP"; other keys → prefix R, e.g. "0.235" → "R0.235") and extensions dict (keys → prefix S, e.g. "0.235" → "S0.235")
  4. Find label with min(abs(open_price - price))
  5. Set trade.near_fib_level = label, trade.fib_distance_pts = distance

_fill_h1_candle(session, trade)

  1. If trade.open_time is None → skip
  2. hour_start = open_time.replace(minute=0, second=0, microsecond=0)
  3. Query current H1 bar: time ∈ [hour_start, hour_start + 1h)
  4. Query previous H1 bar: time ∈ [hour_start - 1h, hour_start)
  5. If no current bar → skip
  6. Build bars = [prev?, current], run detect_pin_bar(bars) → pin_bar_{direction}
  7. Else detect_engulfing(bars) → engulfing_{direction}
  8. Else if open == close → "doji"
  9. Else → "none"

_fill_is_rescue(session, trade)

Count open real trades with same symbol + direction and close_time IS NULL and ticket != trade.ticket. Set is_rescue = count > 0.

Wire-up in trade_logger.py

from services.entry_context import fill_entry_context

# after: session.add(trade); await session.flush()

    await fill_entry_context(session, trade)

4Insight Engine — 5 New Functions

MIN_SAMPLE_SIZE = 5 for all new insight types (lower than existing 10 — fewer tagged trades expected initially). Filter: tagged = [t for t in trades if t.setup_pattern is not None]
4.1 · setup_win_rate

Group tagged closed trades by (setup_pattern, trade_bias, near_fib_level). For each group ≥ 5 trades: compute win_rate + avg_profit. Create one Insight per qualifying group. Deactivate old setup_win_rate insights first.

double_bottom + bullish + near S0.235 → ชนะ 78% (18 เทรด) เฉลี่ย +฿1,240
4.2 · fib_proximity_win_rate

Bucket fib_distance_pts into: close (<5 pts), medium (5–15 pts), far (≥15 pts). Only create insight if any two buckets differ by ≥ 20pp — otherwise not actionable.

Entry ห่าง Fib <5 pts → 74% | 5–15 pts → 51% | >15 pts → 38%
4.3 · rescue_outcome

Split by is_rescue. Both groups need ≥ 5 trades. Compare win rates and avg profits.

ไม้แก้: ชนะ 45% (20 เทรด) vs ไม้เดิม: ชนะ 68% (40 เทรด)
4.4 · best_combo

Group by (setup_pattern, trade_bias, near_fib_level, session). Session derived from open_time converted to ICT (UTC+7) before calling _assign_session(). Report top 3 by win_rate (min 5 each).

Best: morning + double_bottom + bullish + S0.235 → 83% win rate (12 เทรด)
4.5 · post_close_run

Step 1 — Backfill post_close_run_pts for all closed real trades where it's null: query H1 bars [close_time, close_time+8h]. BUY: max(high) − close_price. SELL: close_price − min(low). Store if > 0.

Step 2 — Insight on winning tagged trades (min 3 per pattern).

double_bottom → วิ่งต่อได้ 420 pts | price_cluster → 185 pts หลังคุณปิด

5New Alert Types

Called from run_insight_engine() after insight functions. Cooldown: 24h per alert type — check Alert.sent_at > now − 24h before inserting.
low_winrate_setup

Trigger: (setup_pattern, trade_bias) used ≥ 5 times with win_rate < 0.40

double_top + bullish: ชนะแค่ 28% (9 เทรด) — setup นี้ประวัติไม่ดี พิจารณาใหม่
rescue_ineffective

Trigger: rescue win_rate < 0.35 AND rescue count ≥ 5 AND (initial_win_rate − rescue_win_rate) > 0.20

ไม้แก้ชนะแค่ 30% vs ไม้เดิม 65% — ข้อมูลบอกว่าตัดขาดทุนแล้วเริ่มใหม่ดีกว่า
