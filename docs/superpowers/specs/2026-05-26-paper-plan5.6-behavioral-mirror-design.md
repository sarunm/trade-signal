# Paper Plan 5.6 — Behavioral Mirror (Design Spec)

**Date:** 2026-05-26
**Branch:** `feat/paper-trade-v2`
**Status:** Frozen — ready for implementation plan

## Goal

Make the paper trader close positions in a way that mirrors the user's real
behavior: take profit at roughly the same baht amount they themselves close at,
with a small trailing tolerance so winners are not killed prematurely. Replace
the placeholder Plan 5.5 trail (10% port + signal weakening) with a target
derived from the user's own closed trade history.

## Non-goals

- Mirroring entry behavior (entries still come from pattern discovery / baseline).
- Mirroring exit *signals* (Plan 5.5's pivot/trend/pattern weakening) — dropped
  in favor of a profit-anchored trail. Keeps logic simple, easy to maintain.
- Multi-symbol support — XAUUSD only, same as the rest of the system.
- Adapting per-rule (different rules getting different targets) — one global
  target derived from all winning closed trades.

## Architecture

### New module: `api/services/behavioral_mirror.py`

Single responsibility: compute the user's average winning-trade profit over a
rolling window. Pure read-only query; no side effects.

```
compute_user_avg_profit(
    session: AsyncSession,
    days: int = 30,
    min_sample: int = 10,
) -> Optional[Decimal]
```

Behavior:
- Resolves the current account via `latest AccountSnapshot.account_id`
  (same helper logic as `routers/account.py::_current_account_id`).
- Queries `trades` where `is_paper=False`, `order_state=filled`,
  `close_time >= now - days`, `profit > 0`, and `account_id == current_account`
  (when current_account is not None).
- Returns `Decimal(avg(profit))` quantized to `0.01`.
- Returns `None` if sample size < `min_sample`.
- No caching at this layer — the caller (paper exit manager) caches per tick.

### Modified module: `api/services/paper_exit_manager.py`

Adds a single new branch to `close_paper_trades_on_tick()`:

1. Compute `user_avg` once per tick (cache in a local variable).
2. For each open paper trade with `paper_mode=independent`:
   - First check existing TP/SL exit (unchanged).
   - If no TP/SL hit and `trail_strategy == 'user_avg_trail'` and
     `user_avg is not None`:
     - Compute current `unrealized_profit` using the same side-aware
       price as TP/SL: `tick.bid` for `Direction.buy`, `tick.ask` for
       `Direction.sell`. Reuse the existing `_paper_profit` helper.
     - Read armed state from `recovery_plan['trail']` (an object with
       `peak_profit` and `peak_price`); absent = not armed.
     - If trade has not armed yet:
       - If `unrealized_profit >= user_avg`: arm by setting
         `recovery_plan['trail'] = {'peak_profit': str(unrealized),
         'peak_price': str(price)}`.
     - If armed:
       - If `unrealized_profit > peak_profit`: update `peak_profit` and
         `peak_price` in the same JSON object.
       - If `unrealized_profit <= peak_profit * Decimal('0.80')`: close
         at the side-aware tick price; set
         `paper_exit_reason='user_avg_trail'`.

3. Compute `shadow_profit` once at close time:
   - Simulate "what if we had no trail": project forward to find first hit of
     TP or SL using future price bars (best-effort within current tick window
     only — see Edge Cases for the exact rule).

### Modified table: `paper_trader_rules`

| Change | Column | Type | Notes |
|--------|--------|------|-------|
| ADD | `trail_strategy` | `VARCHAR(30)` | `null` = no trail; `'user_avg_trail'` = enable Plan 5.6 trail. Default `'user_avg_trail'` for new rows. |
| KEEP (deprecated) | `trail_arm_pct` | `NUMERIC(5,4)` | Keep column for now — old rows may have non-null values; ignored by new logic. Removed in a later cleanup migration once all live rules are on `trail_strategy`. |
| KEEP (deprecated) | `trail_enabled` | `BOOLEAN` | Same as above. |

### Modified table: `trades`

| Change | Column | Type | Notes |
|--------|--------|------|-------|
| ADD | `shadow_profit` | `NUMERIC(12,2)` nullable | Profit the paper trade *would have realized* if it ran to TP/SL instead of trailing. Only populated for paper trades closed by `user_avg_trail`. |

## Data flow

```
[real closed trades, profit > 0, last 30d]
           │
           ▼
behavioral_mirror.compute_user_avg_profit(days=30, min_sample=10)
           │  Decimal | None
           ▼
paper_exit_manager.close_paper_trades_on_tick()
           │  per tick: cache user_avg once
           ▼
for each open paper trade (independent mode, trail_strategy=='user_avg_trail'):
    unrealized = mid_price ↔ open_price * volume * 100
    not armed AND unrealized >= user_avg → arm (peak=unrealized)
    armed AND unrealized > peak           → update peak
    armed AND unrealized <= peak * 0.80   → close at current bid/ask
                                            store shadow_profit (TP/SL projection)
                                            paper_exit_reason='user_avg_trail'
```

## Edge cases

| Case | Behavior |
|------|----------|
| < 10 winning real trades in 30d | `compute_user_avg_profit` returns `None`. Paper trades fall back to TP/SL only — no trail behavior. Logged once per tick at `WARNING`. |
| Real trade history is empty | Same as above. |
| user_avg changes mid-trade (rolling window shifts) | The armed `peak_profit` is locked once stored in `recovery_plan`. Re-arming uses the *current* tick's `user_avg`, but never reverses an existing armed state. |
| Tick has no `bid`/`ask` separation (degenerate data) | Skip the trade — same defensive pattern as existing TP/SL logic. |
| Multiple ticks in the same second | Each tick independently checks arm/peak/exit. No state cleanup needed. |
| Shadow projection: future TP/SL | Computed at close time using the *latest* H1 bar high/low to estimate "would TP/SL have hit by now". If neither, set `shadow_profit = unrealized_profit_at_close` (i.e., assume the position would have stayed open and we're sampling at the same moment). This is a simple, conservative approximation; the gap analysis is suggestive rather than authoritative. |
| `trail_strategy IS NULL` | Skip trail logic entirely. Equivalent to old TP/SL-only behavior. |
| Manual override / paper rule with `mode='strict'` | Same trail strategy applies regardless of `mode`. Strategy is per-rule, not per-mode. |

## Configuration

Environment variables (with defaults):

| Var | Default | Purpose |
|-----|---------|---------|
| `BEHAVIORAL_MIRROR_DAYS` | `30` | Rolling window for `compute_user_avg_profit`. |
| `BEHAVIORAL_MIRROR_MIN_SAMPLE` | `10` | Minimum winning trades to return a value. |
| `BEHAVIORAL_TRAIL_RETRACE_PCT` | `0.20` | Retrace fraction from peak that triggers close. |

These live in `paper_exit_manager.py` (read once at module load, like other
config in the project). Override only via `.env`.

## Testing

All tests use the existing `tests/` SQLite + async pytest setup.

### `tests/test_behavioral_mirror.py` (new)
- Returns None when no real winning trades exist.
- Returns None when below min_sample.
- Computes correct avg from winning trades within window.
- Excludes losing trades, paper trades, and trades outside window.
- Excludes trades from other accounts (filters on the same `account_id` rule
  used by `daily-pl`).

### `tests/test_paper_exit_manager.py` (extend existing)
- Trade with `trail_strategy=null` → no trail behavior, falls back to TP/SL.
- Trade with `trail_strategy='user_avg_trail'`, no real history → no trail.
- Arms when `unrealized >= user_avg`, peak stored in `recovery_plan`.
- Updates peak on higher unrealized.
- Closes when `unrealized <= peak * 0.80`, sets `paper_exit_reason='user_avg_trail'`.
- `shadow_profit` populated on close (mocked H1 bar projection).
- TP/SL still wins over trail when both could trigger on same tick.

## Out of scope (deferred)

- Per-pattern targets (different patterns may have different ideal exits).
- Volatility-aware retrace (20% may be too tight in trending markets).
- Time-of-day bias in `user_avg` (different sessions may have different
  averages).
- Lot-sizing mirror (mirroring user's typical volume, not just exit point).
- Closing the loop: surfacing shadow vs actual gap in dashboard analytics
  (Plan 5.7 candidate).
